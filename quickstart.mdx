This guide will help you convert your first Minecraft world between Java and Bedrock editions using Chunker.

## Prerequisites

Before you begin, ensure you have:

- Java 17 or higher installed
- A Minecraft world to convert (backed up!)
- At least 4GB of available RAM

<Note>
Always backup your worlds before conversion to prevent any data loss.
</Note>

## Choose your method

<Tabs>
  <Tab title="Desktop App">
    ## Desktop application quickstart

    <Steps>
      <Step title="Download and install">
        Download the latest Chunker desktop application from the [GitHub releases page](https://github.com/HiveGamesOSS/Chunker/releases).

        <CodeGroup>
        ```bash Windows
        # Download Chunker.exe
        # Double-click to run
        ```

        ```bash macOS
        # Download Chunker.dmg
        # Drag to Applications folder
        ```

        ```bash Linux
        # Download Chunker.AppImage
        chmod +x Chunker.AppImage
        ./Chunker.AppImage
        ```
        </CodeGroup>
      </Step>

      <Step title="Select your world">
        1. Click **Select World** in the application
        2. Navigate to your Minecraft world folder:
           - **Java**: `.minecraft/saves/WorldName`
           - **Bedrock**: Check your game's storage location
        3. Select the world folder (or drag and drop)
      </Step>

      <Step title="Choose output format">
        1. Click **Export As**
        2. Select the target edition and version
           - For Java → Bedrock: Select a Bedrock version (e.g., 1.20.80)
           - For Bedrock → Java: Select a Java version (e.g., 1.20.5)
        3. Click **Continue**
      </Step>

      <Step title="Start conversion">
        1. Review advanced settings if needed (optional)
        2. Click **Start Conversion**
        3. Wait for the progress bar to complete
        4. Choose where to save your converted world
      </Step>
    </Steps>

  <Tab title="CLI">
    ## Command-line quickstart

    <Steps>
      <Step title="Download the CLI">
        Download the latest `chunker-cli.jar` from the [GitHub releases page](https://github.com/HiveGamesOSS/Chunker/releases).

        ```bash
        wget https://github.com/HiveGamesOSS/Chunker/releases/latest/download/chunker-cli.jar
        ```
      </Step>

      <Step title="Prepare your world">
        Locate your Minecraft world directory:

        ```bash
        # Java Edition (example paths)
        # Windows: C:\Users\YourName\AppData\Roaming\.minecraft\saves\WorldName
        # macOS: ~/Library/Application Support/minecraft/saves/WorldName
        # Linux: ~/.minecraft/saves/WorldName

        # Bedrock Edition (example paths)
        # Windows: C:\Users\YourName\AppData\Local\Packages\Microsoft.MinecraftUWP_8wekyb3d8bbwe\LocalState\games\com.mojang\minecraftWorlds\WorldName
        # Android: /storage/emulated/0/games/com.mojang/minecraftWorlds/WorldName
        ```

        <Warning>
        Make a backup of your world before conversion!
        </Warning>
      </Step>

      <Step title="Run the conversion">
        Execute Chunker with your input world, desired output format, and output directory.

        <CodeGroup>
        ```bash Java to Bedrock
        java -jar chunker-cli.jar \
          -i "/path/to/java/world" \
          -f BEDROCK_1_20_80 \
          -o "converted_world_bedrock"
        ```

        ```bash Bedrock to Java
        java -jar chunker-cli.jar \
          -i "/path/to/bedrock/world" \
          -f JAVA_1_20_5 \
          -o "converted_world_java"
        ```

        ```bash Upgrade Java version
        java -jar chunker-cli.jar \
          -i "my_world_1.12" \
          -f JAVA_1_21_0 \
          -o "my_world_1.21"
        ```
        </CodeGroup>

        The conversion will display progress updates:

        ```
        Converting from Java 1.12.2 to Bedrock 1.20.80
        0.00%
        25.50%
        50.75%
        75.25%
        100.00%
        Conversion complete! Took 0h 2m 35s 420ms
        ```
      </Step>

      <Step title="Use your converted world">
        Your converted world is now in the output directory and ready to use!

        - **Java worlds**: Copy to `.minecraft/saves/`
        - **Bedrock worlds**: Copy to your game's worlds directory
      </Step>
    </Steps>

    </Card>

</Tabs>

## Available formats

To see all available output formats, run:

```bash
java -jar chunker-cli.jar -f ?
```

This will list all supported versions in the format `EDITION_VERSION`, such as:
- `JAVA_1_20_5`
- `BEDROCK_1_20_80`
- `JAVA_1_21_0`

<Accordion title="Common output formats">
**Java Edition**
- `JAVA_1_20_6` - Latest 1.20.x
- `JAVA_1_21_0` - Minecraft 1.21
- `JAVA_1_19_4` - Legacy 1.19
- `JAVA_1_18_2` - Legacy 1.18

**Bedrock Edition**
- `BEDROCK_1_20_80` - Current stable
- `BEDROCK_1_21_0` - Minecraft 1.21
- `BEDROCK_1_19_80` - Legacy 1.19
</Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Out of memory error">
    Increase Java heap size when running Chunker:

    ```bash
    # Desktop app
    Chunker.exe -Xmx8G

    # CLI
    java -Xmx8G -jar chunker-cli.jar -i "world" -f BEDROCK_1_20_80 -o "output"
    ```
  </Accordion>

  <Accordion title="World not detected">
    Make sure you're selecting the world **folder** (not a region file or level.dat):
    - Java: The folder containing `level.dat` and `region/` directory
    - Bedrock: The folder containing `level.dat` and `db/` directory
  </Accordion>

  <Accordion title="Conversion takes too long">
    Large worlds can take significant time. Consider:
    - Using pruning to convert only specific regions
    - Increasing available memory
    - Running on a machine with better performance
    - Using an SSD instead of HDD

    See [pruning documentation](/advanced/pruning) for region selection.
  </Accordion>

  <Accordion title="Missing blocks or items">
    Some blocks and items don't have direct equivalents between editions. Chunker uses best-effort mappings, but some content may:
    - Convert to similar blocks
    - Convert to placeholder blocks
    - Be removed if no mapping exists

    See [supported versions](/usage/supported-versions) for edition differences.
  </Accordion>
</AccordionGroup>
